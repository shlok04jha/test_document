Supported Operations¶

Arithmetic¶

  • ADD - Adds two operands together.

    Example: 10 + 5 = 15

  • SUB - Subtracts the second operand from the first.

    Example: 10 - 5 = 5

Logical¶

  • AND - Performs bitwise AND operation.

    Example: 1100 AND 1010 = 1000

  • OR - Performs bitwise OR operation.

    Example: 1100 OR 1010 = 1110

  • XOR - Performs bitwise Exclusive OR operation.

    Example: 1100 XOR 1010 = 0110

Shift¶

  • SLL - Shifts bits to the left logically.

    Example: 0001 << 2 = 0100

  • SRL - Shifts bits to the right logically and fills zeros.

    Example: 1000 >> 2 = 0010

  • SRA - Shifts bits to the right while preserving the sign bit.

    Example: 11111000 >> 2 = 11111110

Comparison¶

  • EQ - Checks if two operands are equal.

    Example: 5 == 5 → True

  • NE - Checks if two operands are not equal.

    Example: 5 != 3 → True

  • GT - Checks if the first operand is greater than the second.

    Example: 8 > 5 → True

  • GE - Checks if the first operand is greater than or equal to the second.

    Example: 5 >= 5 → True

  • LT - Checks if the first operand is less than the second.

    Example: 3 < 5 → True

  • LE - Checks if the first operand is less than or equal to the second.

    Example: 5 <= 5 → True

  • SLT - Sets result if the first signed operand is less than the second.

    Example: -2 < 3 → True

  • SLE - Sets result if the first signed operand is less than or equal to the second.

    Example: -2 <= -2 → True
